# Jaz widget style guide

Maintained by Jaz — do not edit this file; Jaz overwrites it. `index.html` next to it is your widget: an HTML **fragment** that Jaz wraps in a host document (design tokens, Tailwind, a message bridge) and renders in a sandboxed tile on the user's board. Read this before editing the widget.

## Contract

- Fragment only: no `<!doctype>`, `<html>`, `<head>`, or `<body>`. Inline `<style>` and `<script>` are allowed. Keep it under 1 MB.
- External `<script src>` will NOT load (CSP allows only inline JS and the host's own assets). `fetch()` to https endpoints works at render time, but bake the current data into the markup each run so the tile renders offline.
- Links open in the user's browser. The widget is read-only: present information; never render forms or buttons that promise actions.
- Show ONLY the data the user asked the loop to track. The tile chrome already shows the title and freshness — no repeated headings, no "as of …" or "data from …" footers, no meta commentary.

## Layout: fill the tile

The tile is a fixed box; trailing empty space looks broken. Design for the full height:

- Use ONE root element. The host stretches it to the tile's height automatically.
- Make the root a vertical flex (`jz-stack` or `flex flex-col`). Fixed parts (KPI row, summary) go on top; give the main content area `jz-fill jz-scroll` (Tailwind: `flex-1 min-h-0 overflow-y-auto`) so it grows to the bottom edge and scrolls internally when long.
- Keep KPIs and headings visible; scroll the list, not the page. Never set a fixed pixel height on the root.
- The user can resize the tile wider than your size_hint — rows must still read at full board width. A lone value hugging the far right edge of a wide row reads as dead space; keep metadata adjacent to the text it describes, and right-align only true value columns.
- The host caps the root at the tile height ONLY when it contains a `jz-scroll`/`overflow-y-auto` scroller; otherwise the root keeps its natural height and the tile scrolls as a whole. So: long content → designate one scroller. Without one, never assume extra vertical room exists.
- Content must never be squeezed or cropped to fit. Avoid `grid-template-rows: 1fr`-style tracks combined with fixed-size or `aspect-ratio` content (clock faces, gauges, album art) — when the tile is short, the tracks compress and the content gets clipped. Bound such visuals by the available height too (`max-height`, `min()` with a height term) or let the container scroll.

```html
<div class="jz-stack">                            <!-- stretched to the tile -->
  <div class="jz-kpis">…</div>                    <!-- fixed top -->
  <ul class="jz-list jz-fill jz-scroll">…</ul>    <!-- grows + scrolls -->
</div>
```

## Color: copy the Jaz palette

The host theme switches light/dark automatically — the tokens flip, your markup doesn't change. Never hardcode hex/rgb colors; Tailwind's stock palette (`red-500`, `slate-200`, …) is disabled on purpose.

| Token | Tailwind utility | Use for |
| --- | --- | --- |
| `--color-surface` | `bg-surface` | the tile background (already set on the page) |
| `--color-surface-2` | `bg-surface-2` | inset panels, cards, progress tracks |
| `--color-ink` | `text-ink` | primary text |
| `--color-ink-2` | `text-ink-2` | secondary text, labels |
| `--color-ink-3` | `text-ink-3` | faint text, timestamps |
| `--color-border` | `border-border`, `divide-border` | hairlines, dividers |
| `--color-primary` | `bg-primary`, `text-primary` | THE accent (cobalt). One accent per widget. |
| `--color-primary-soft` | `bg-primary-soft` | tinted chip/row backgrounds |
| `--color-primary-strong` | `text-primary-strong` | accent text on soft backgrounds |
| `--color-on-primary` | `text-on-primary` | text on solid primary |
| `--color-accent` / `--color-accent-soft` | `text-accent`, `bg-accent-soft` | warm highlight, sparingly |
| `--color-ok` / `--color-running` / `--color-danger` (+ `danger-soft`) | `text-ok`, `bg-danger-soft`, … | status ONLY — never decoration |

Rules of thumb: hierarchy comes from the ink scale and surface layering, not from extra colors. Cobalt is the single accent; status colors appear only when they mean status. Charts: use `--color-primary` for the main series, `--color-border` for muted bars, `--color-accent` for one contrast series.

## Type

Inter is loaded. The Tailwind scale is remapped to the Jaz sizes — use it instead of arbitrary values: `text-xs` 11px (labels, timestamps), `text-sm` 12px (secondary), `text-base` 13px (body, default), `text-lg` 15px / `text-xl` 18px / `text-2xl` 22px (key numbers). Weights: 400–500 text, 600–650 for values and headings. Numbers always get `tabular-nums` (class `jz-mono` for ids/hashes/code only).

## Radius

Small radii look cheap. The Tailwind scale is remapped: `rounded-sm` 8px, `rounded-md` 10px, `rounded-lg`/`rounded-card` 12px, `rounded-xl` 16px, `rounded-full` for pills and dots. Visible boxes never go below 8px. Nested radii are concentric: inner = outer − padding (card 12 with 4px padding → inner chip 8); the same radius on parent and child is the most common thing that makes a tile feel off.

## Lists: pick ONE row style

Lists are the most common widget shape; this is where designs go ugly. Choose exactly one of these and stay consistent:

- **Flat rows** — `jz-list` + `jz-item`: hairline dividers, no row backgrounds. Densest; the default for 5+ rows.
- **Chip rows** — `flex flex-col gap-1` + rows of `rounded-md bg-surface-2 px-2.5 py-1.5`: no `jz-item`, no dividers (chips + hairlines together look broken).

Canonical ranked row (flat style — note: rank fixed-width and faint, title truncates, value right-aligned and tabular):

```html
<ul class="jz-list jz-fill jz-scroll">
  <li class="jz-item">
    <span class="w-5 shrink-0 text-xs tabular-nums text-ink-3">1</span>
    <img class="size-4 shrink-0 rounded-[4px]" src="…" alt="" />
    <span class="jz-item-title">NVIDIA</span>
    <span class="jz-item-value">$4.93T</span>
  </li>
</ul>
```

Match the row style to the data:

- **Name + number data** (rankings, prices, counts) → flat rows. Don't add a second text line unless it carries real information — a ticker/timestamp usually belongs inline after the title in `text-xs text-ink-3`, not on its own line.
- **Feeds of sentences** (tweets, headlines, jokes, commit messages) are not ranked lists — the text IS the content. Use chip rows with the text as the body (`text-pretty`, `line-clamp-2` so long posts wrap to two lines instead of truncating to nothing) and one `text-xs text-ink-3` metadata line under it (time · count, with a small inline SVG icon to label what the count is — a bare "3.1k" means nothing).
- Rows that link somewhere should show it on hover: tint the chip (`hover:bg-primary-soft`) or underline the title, transitioning only the property that changes (`transition-colors`), never `transition-all`.

Don't render a table-style header row over a list; if columns need labels, use a real `jz-table`.

## Images

- Hotlink only URLs you verified THIS run (fetched them and got an image back). The board hides images that fail to load and reports the count in your next run's prompt — but a hidden image still leaves a hole, so a guessed URL is never worth it.
- An image that is identical on every row (the same avatar on each tweet, the same logo on each item) carries no information — show it once or drop it entirely.
- Give photos, logos, and avatars a hairline outline so they sit cleanly on any surface: `outline outline-1 -outline-offset-1 outline-black/10 dark:outline-white/10`.

## Components (jz-*)

Prebuilt pieces that already match the app — compose these before inventing your own. They are defaults, not locks: Tailwind utilities on the same element override them (e.g. `jz-card p-2` tightens the padding).

- Layout: `jz-stack` (column, 8px gap), `jz-row`, `jz-spread` (space-between), `jz-card` (inset panel), `<hr class="jz-divider">`, `jz-fill` (grow), `jz-scroll` (internal scroll).
- Text: `jz-muted`, `jz-faint`, `jz-small`, `jz-mono`.
- KPIs: `<div class="jz-kpis"><div class="jz-stat"><div class="jz-stat-value">42</div><div class="jz-stat-label">Unread</div><div class="jz-stat-delta down">-3</div></div></div>`
- List: `<ul class="jz-list"><li class="jz-item"><span class="jz-item-title">Fix login bug</span><span class="jz-item-sub">2h ago</span><span class="jz-item-value">P1</span></li></ul>`
- Table: `<table class="jz-table">` with plain `<th>`/`<td>`.
- Badge: `<span class="jz-badge ok">passing</span>` (variants: `primary`, `accent`, `ok`, `running`, `danger`).
- Progress: `<div class="jz-progress"><div class="jz-progress-fill" style="width:62%"></div></div>`
- Bars: `<div class="jz-bars"><div class="jz-bar" style="height:62%"></div>…</div>` (`accent`, `muted` variants).
- Empty state: `<div class="jz-empty">Nothing yet</div>`
- Inline SVG is welcome for custom visuals — use the tokens for fills/strokes.

## Tailwind

Tailwind v4 utilities compile inside the tile (no build step), themed to the tokens above. Use them freely for layout and spacing: `flex items-baseline justify-between gap-2`, `grid grid-cols-2 gap-2`, `p-3`, `truncate`, `tabular-nums`. Light/dark is handled by the tokens, so you almost never need `dark:` variants.

## Quality bar

Before publishing, check: the widget fills the tile with no dead space at the bottom; nothing is squeezed, overlapping, or cropped; long lists scroll internally; long titles `truncate` (feed text `line-clamp-2` instead); numbers are tabular, labeled, and right-aligned in lists/tables; spacing follows a 4/8/12px rhythm; row style is consistent (flat OR chips, never mixed); at most one accent color; no box-in-box-in-box nesting; every image URL was verified this run and no image repeats on every row; empty data shows `jz-empty`, not a blank tile.

## Publishing

Publish with the `publish_widget` tool or the `_jaz.dev/widget/publish` extension method if available (it validates and returns errors); otherwise the file auto-publishes when the run ends. The publish result may carry lint warnings (hardcoded colors, viewport units, disabled palette classes, …) — fix them and republish in the same run. The board also measures the rendered result (dead space, overflow, clipped content) and reports problems in your next run's prompt. Pass `size_hint` "WxH" in grid tiles (W 1–6, H 1–8; 2x2 small, 3x2 medium, 6x3 full-width; one row ≈ 120px). Size the tile to the content you actually render — a fixed list of N rows wants a tile that N rows fill, not a taller one with dead space under it. The user can move and resize the tile afterwards; their layout always wins. Runtime JS errors from the board are reported back to you on the next run — fix them first.
