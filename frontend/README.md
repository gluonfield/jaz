# jaz desktop

Electron desktop app for the jaz backend. React + TypeScript + TanStack Router (file-based SPA) + TanStack Query + Tailwind v4, built with electron-vite. Bun is the package manager and script runner; the Electron main process runs on Electron's bundled Node.

## Run

```bash
# 1. backend (from ../backend) â€” or skip this and click "Start locally"
#    on the launch screen, which runs it for you
go run ./cmd/jaz

# 2. desktop app
bun install
bun run dev
```

Point the app at a different backend with `JAZ_API_URL` (an explicit value
beats the URL remembered from the last session):

```bash
JAZ_API_URL=http://127.0.0.1:18080 bun run dev
```

## Scripts

- `bun run dev` â€” dev mode with HMR (vite on port 5180)
- `bun run build` â€” distributable: compiles the Go backend, bundles, and packages a signed `Jaz.app` + dmg into `dist/` via electron-builder (alias of `build:app`; notarizes when Apple credentials are in the env)
- `bun run build:bundle` â€” just the electron-vite production bundle into `out/`
- `bun run typecheck` â€” renderer + main/preload TypeScript

## Release

GitHub Actions builds standalone CLI tarballs for Linux/macOS and signed macOS desktop artifacts when a GitHub Release is published in `gluonfield/jaz`. The CLI assets are named `jaz-server_<version>_<os>_<arch>.tar.gz` for the bare `jaz` server binary and `jaz-chat_<version>_<os>_<arch>.tar.gz` for the TUI client. The desktop job attaches the DMG, ZIP, blockmaps, and `latest-mac.yml` to that release. Add these repository secrets:

- `APPLE_CERTIFICATE_P12_BASE64`
- `APPLE_CERTIFICATE_PASSWORD`
- `APPLE_API_KEY_P8_BASE64`

Add these repository variables:

- `APPLE_TEAM_ID`
- `APPLE_API_KEY_ID`
- `APPLE_API_ISSUER`

The release tag must match `frontend/package.json`, for example `v0.0.2`.

## Layout

- `src/main/` â€” Electron main process (window, titlebar style)
- `src/preload/` â€” context bridge; exposes `window.jaz.apiBaseUrl`
- `src/renderer/src/routes/` â€” file-based routes (`__root` shell, `/`, `/agent`, `/sessions`, `/sessions/$sessionId`)
- `src/renderer/src/lib/api/` â€” typed client for the Go REST + SSE API
- `src/renderer/src/styles/globals.css` â€” design tokens (see `../DESIGN.md`)

## Notes

- `routeTree.gen.ts` is generated by the TanStack Router vite plugin; don't edit it.
- Agent files (AGENTS.md, SOUL.md) are read and written through the backend (`/v1/agent/files`), never the local filesystem.
- Session events stream over SSE (`EventSource`); the backend names its frames (`event: <type>`), so new event types must be registered in `lib/api/sse.ts`.
- `trustedDependencies` in package.json lets bun run Electron's postinstall (binary download). If `node_modules/electron/dist` is missing after install, run `cd node_modules/electron && node install.js`.
